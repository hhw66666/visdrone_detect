## 一、目前已完成的工作内容与进度执行情况

### 1.1 项目概述

本项目为**基于 YOLOv8 的智能航拍目标检测系统**，命名为"科大巡弋"，旨在为无人机航拍提供高精度、高效率的目标识别解决方案。系统采用 Django 框架构建 Web 界面，支持图片检测、视频检测、历史记录管理、AI 客服等功能。

### 1.2 完成功能模块

| 模块 | 功能描述 | 状态 | 技术实现 |
|------|----------|------|----------|
| **用户认证系统** | 注册、登录、退出 | ✅ 已完成 | Django Auth + AJAX 弹窗登录 |
| **图片检测** | 单张图片快速检测 | ✅ 已完成 | YOLOv8 + OpenCV |
| **视频检测** | 视频文件逐帧检测 | ✅ 已完成 | YOLOv8 + OpenCV |
| **历史记录** | 检测记录查看、删除 | ✅ 已完成 | Django ORM + SQLite |
| **AI 客服** | 智能问答、天气查询、地点搜索 | ✅ 已完成 | MiniMax M2.7 + ModelScope MCP |
| **聊天历史** | 对话记录保存、管理 | ✅ 已完成 | AJAX + Django ORM |
| **个人资料** | 头像上传、用户信息管理 | ✅ 已完成 | Django FileField |

## 二、个人毕业设计工作体会

### 2.1 工作量大小

本项目从零开始构建了一个完整的智能航拍目标检测系统，包含：

- **后端代码**：约 2500+ 行 Python 代码（Django views、models、config）
- **前端代码**：约 2000+ 行 HTML/CSS/JavaScript（7个页面 + 公共组件）
- **AI 服务**：约 500+ 行 Python 代码（Agent、MCP、提示词）
- **配置文件**：config.xml、settings.py、urls.py 等
- **文档**：README.md、项目进度报告

**总代码量估算**：约 5000+ 行

### 2.2 难度分析

本项目涉及多个技术领域的整合，具有一定的技术难度：

| 技术难点 | 描述 | 解决方案 |
|----------|------|----------|
| **YOLOv8 集成** | 需要在 Django 中调用 YOLO 模型进行图片/视频检测 | 使用 ultralytics 库，异步加载模型 |
| **视频帧处理** | 视频检测需要逐帧处理并合并结果 | 使用 OpenCV 视频读取，批量推理 |
| **AI 模型调用** | MiniMax API 调用需要处理流式响应 | 使用 SSE 或轮询方式获取完整回复 |
| **MCP 工具集成** | ModelScope MCP 连接高德地图 API | 实现 streamable_http 类型的 MCP 客户端 |
| **弹窗登录流程** | AJAX 登录 + 模态框 + 页面跳转 | fetch API + window.location.href 跳转 |
| **中文显示** | YOLO 检测框绘制中文标签 | 使用 PIL ImageDraw 绘制中文 |

## 三、存在的问题与解决方案

### 3.1 模型精度问题（核心问题）

| 问题 | 描述 | 解决方案 |
|------|------|----------|
| **模型精度不足** | 当前 YOLOv8 模型在复杂航拍场景下检测精度有限，具体表现为：<br>1. **光照变化**：强光、逆光、夜间环境下误检率高<br>2. **目标遮挡**：目标被部分遮挡时容易漏检<br>3. **小目标检测**：远处的小目标检测效果差<br>4. **密集场景**：多目标聚集时容易出现重复检测和漏检<br>5. **背景干扰**：与目标相似的背景物体容易被误检 | **短期改进：**<br>1. 调优置信度阈值（conf）和 NMS 阈值（iou），找到最佳平衡点<br>2. 对输入图像进行预处理（直方图均衡化、对比度增强）<br><br>**中期改进：**<br>3. 收集更多复杂场景的航拍数据，重点补充夜间、逆光、遮挡等样本<br>4. 使用数据增强策略（mosaic、mixup、旋转、翻转、色彩抖动）提升泛化能力<br>5. 对小目标进行 oversampling 和 multi-scale training<br><br>**长期改进：**<br>6. 尝试 YOLOv11、YOLOv12 等更先进的目标检测模型<br>7. 使用模型融合或多模型集成（如 YOLOv8 + YOLOv10）<br>8. 引入注意力机制（Attention）提升对遮挡和复杂场景的鲁棒性<br>9. 使用 Transformer 架构的检测器（如 RT-DETR）<br>10. 考虑使用半监督学习利用未标注数据 |

### 3.2 技术问题

| 问题 | 描述 | 解决方案 |
|------|------|----------|
| **模型推理速度** | YOLOv8 在 CPU 上推理较慢，大图或视频处理耗时明显 | 1. 使用 GPU 加速推理<br>2. 对模型进行量化（INT8）加速<br>3. 使用模型剪枝减少计算量<br>4. 对视频进行采样检测而非逐帧检测 |
| **视频检测内存** | 长时间处理大视频可能导致内存溢出 | 1. 添加视频大小限制（最大分辨率和时长）<br>2. 分段处理视频，释放内存<br>3. 使用生成器流式处理帧 |
| **并发访问** | Django 开发服务器不适合高并发场景 | 生产环境使用 gunicorn + nginx 部署 |
| **文件上传安全** | 未对上传文件类型做严格限制 | 添加文件头检测、MIME 类型校验、文件大小限制 |

### 3.3 功能问题

| 问题 | 描述 | 解决方案 |
|------|------|----------|
| **登录状态过期** | Session 过期后 AJAX 请求返回 401 但未处理 | 添加 401 状态码处理，自动跳转登录页 |
| **聊天记录重复保存** | 关闭页面时可能保存多条重复记录 | 添加去重逻辑，根据时间戳和内容判断 |
| **图片旋转问题** | 部分手机拍摄图片有 EXIF 旋转信息，检测结果位置偏移 | 使用 PIL 处理 EXIF 信息后再进行检测 |
| **视频检测进度** | 视频检测耗时长，用户无法得知进度 | 添加进度条显示，支持后台任务处理 |

### 3.4 界面问题

| 问题 | 描述 | 解决方案 |
|------|------|----------|
| **移动端适配** | 部分页面在极窄屏幕上布局异常 | 添加更多响应式断点，优化移动端布局 |
| **深色模式** | 目前仅支持深色主题 | 可后续添加亮色主题切换功能 |
| **加载状态提示** | 检测过程较久时无加载动画 | 添加 skeleton 屏和加载动画提升体验 |

### 3.5 部署问题

| 问题 | 描述 | 解决方案 |
|------|------|----------|
| **跨局域网访问** | 校园网可能阻止设备间互访 | 使用内网穿透工具（如 ngrok、frp） |
| **HTTPS** | 当前为 HTTP 明文传输，存在安全风险 | 生产环境配置 SSL 证书 |
| **数据库备份** | SQLite 不适合高并发写入 | 可迁移到 PostgreSQL |
| **模型文件管理** | 模型文件较大，版本管理困难 | 使用 DVC 或单独的对象存储 |

### 3.6 后续改进方向

1. **性能优化**
   - 添加 Redis 缓存检测结果
   - 使用 Celery 异步任务队列处理视频检测
   - 模型量化或剪枝加速推理

2. **功能扩展**
   - 添加视频流实时检测（RTSP 推流）
   - 支持更多目标类别
   - 添加数据导出功能（Excel、JSON）
   - 支持截图粘贴上传

3. **AI 能力增强**
   - 接入更多 MCP 工具（翻译、计算器、知识查询等）
   - 添加 RAG 知识库实现专业问答
   - 支持多轮对话上下文记忆

4. **安全加固**
   - 添加登录失败次数限制，防止暴力破解
   - 添加图片内容安全审核
   - 防止 SQL 注入和 XSS 攻击
   - 添加操作日志审计

## 四、总结

本项目基本完成了智能航拍目标检测系统的全部核心功能，涵盖了用户认证、目标检测、AI 客服、历史管理等多个模块。系统界面美观，功能完整，具备一定的实用价值。

在开发过程中，遇到了 Django 与 YOLO 集成、AI 模型调用、MCP 工具对接等技术难点，通过查阅文档和实践探索均已解决。

**当前最核心的问题是模型精度不足**，尤其是在复杂航拍场景下（光照变化、目标遮挡、小目标等）。这是目标检测领域的常见挑战，需要通过数据增强、模型升级、参数调优等多方面努力来逐步改善。

后续将继续完善文档，优化性能，提升用户体验，并进行实际的航拍数据测试验证系统效果，为毕业论文提供充足的实验数据支撑。
